# Preparación para open source

Usa esta lista antes de publicar o compartir un repositorio de ingeniería asistida por IA.

- [ ] Sin secretos
- [ ] Sin credenciales
- [ ] Sin tokens
- [ ] Sin API keys
- [ ] Sin nombres de clientes
- [ ] Sin presupuestos
- [ ] Sin detalles contractuales
- [ ] Sin URLs privadas
- [ ] Sin capturas privadas
- [ ] Sin prompts sensibles
- [ ] Revisar workflows
- [ ] Revisar `.env.example`
- [ ] Revisar dependencias
- [ ] Revisar licencias
- [ ] Revisar documentación generada
- [ ] Revisar contenido vendorizado

## Por qué importa

La preparación para open source no trata solo de calidad técnica. También protege a las personas, los contratos, la infraestructura y la confianza.
