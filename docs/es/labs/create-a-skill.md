# Laboratorio: Crear una skill

## Objetivo

Empaquetar una tarea repetida de ingeniería como una skill duradera.

## Escenario

Con frecuencia necesitas la misma secuencia para actualizar docs y validar el resultado.

## Pasos

1. Nombra la skill.
2. Define su trigger y scope.
3. Lista inputs y outputs esperados.
4. Agrega validación y failure modes.
5. Incluye un ejemplo pequeño.

## Resultado esperado

Una skill reutilizable que distintas tareas puedan invocar de forma consistente.

## Preguntas de reflexión

- ¿Qué parte de la tarea es estable?
- ¿Qué parte todavía necesita juicio humano?

## Notas de seguridad

Mantén la skill segura para uso público y evita codificar contexto sensible del proyecto.
