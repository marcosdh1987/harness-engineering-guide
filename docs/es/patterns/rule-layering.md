# Capas de reglas

Organiza las reglas en capas para que cada una tenga un trabajo claro.

- **Reglas globales** definen valores por defecto personales o de la organización.
- **Reglas del repositorio** capturan restricciones y workflows de todo el proyecto.
- **Reglas de dominio** explican áreas especializadas como ML, documentación o despliegue.
- **Reglas específicas de tarea** reducen el alcance para un solo cambio.
- **Adapters específicos por herramienta** reformatean la misma intención para cada herramienta de IA.

Las capas reducen conflictos y facilitan actualizar la fuente de verdad.
