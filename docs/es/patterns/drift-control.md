# Control de drift

El drift aparece cuando reglas, skills, docs, adapters y automatización dejan de coincidir entre sí.

## Por qué importa

El drift hace que las herramientas de IA sean menos confiables y que las revisiones humanas sean más lentas. Las personas dejan de confiar en el proceso documentado cuando los artefactos generados ya no coinciden con la intención fuente.

## Cómo reducirlo

Usa verificaciones de CI, comandos de sincronización, revisiones de documentación y pequeños ciclos de mantenimiento para mantener alineado el harness.
