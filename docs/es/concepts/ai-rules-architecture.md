# Arquitectura de reglas de IA

Un harness práctico puede describirse como cuatro niveles.

1. **Governance** — políticas duraderas, restricciones de arquitectura, límites de seguridad y reglas para compartir en público.
2. **Operational Skills** — capacidades repetibles para tareas comunes como editar docs, ejecutar validación o preparar releases.
3. **Automation** — jobs de CI, formato, tests, verificaciones de sincronización y otras puertas mecánicas.
4. **Orchestration** — workflows de plan primero que deciden cuándo aplicar reglas, skills y automatización.

Los cuatro niveles deben reforzarse entre sí. Governance fija límites, las skills facilitan el trabajo común, la automatización verifica resultados y la orquestación mantiene disciplina en la ejecución.
