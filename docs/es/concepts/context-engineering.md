# Context Engineering

La calidad del contexto afecta fuertemente el desarrollo asistido por IA.

## Cómo se ve un buen contexto

Un buen contexto incluye estructura clara del repositorio, documentación útil, reglas duraderas, ejemplos concretos y tests que muestran el comportamiento esperado.

## Por qué el contexto obsoleto es peligroso

El contexto puede desviarse cuando la documentación, las reglas generadas, los ejemplos y la automatización dejan de coincidir con el repositorio real. Entonces las herramientas de IA toman decisiones desactualizadas con demasiada confianza.

## Cómo reducir la ambigüedad

- Mantén la documentación cerca del código o workflow que describe.
- Prefiere ejemplos que coincidan con la práctica actual.
- Elimina o actualiza instrucciones obsoletas.
- Haz que los pasos de validación sean fáciles de encontrar y ejecutar.
