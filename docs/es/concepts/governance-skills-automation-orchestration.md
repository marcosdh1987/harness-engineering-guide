# Governance, skills, automation y orchestration

## Governance

Governance define políticas duraderas y restricciones arquitectónicas. Algunos ejemplos son las reglas de seguridad para open source, los requisitos de revisión, las políticas de dependencias y los estándares de documentación.

## Operational skills

Las operational skills empaquetan capacidades repetibles. Una skill puede guiar a un agente para actualizar documentación, validar un cambio pequeño o preparar un ejemplo público seguro.

## Automation

Automation ejecuta verificaciones mecánicas confiables. Ejemplos comunes son builds de CI, formato, tests, validación de enlaces y verificaciones de sincronización para artefactos generados.

## Orchestration

Orchestration conecta los niveles en un método de trabajo. Un workflow de plan primero, ediciones enfocadas, validation gates y disciplina explícita de revisión ayudan a que las herramientas de IA se comporten como parte de un sistema y no como asistentes aislados.
