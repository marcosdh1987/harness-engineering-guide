# Open-Source Readiness

Use this checklist before publishing or sharing an AI-assisted engineering repository.

- [ ] No secrets
- [ ] No credentials
- [ ] No tokens
- [ ] No API keys
- [ ] No client names
- [ ] No budgets
- [ ] No contractual details
- [ ] No private URLs
- [ ] No private screenshots
- [ ] No sensitive prompts
- [ ] Check workflows
- [ ] Check `.env.example`
- [ ] Check dependencies
- [ ] Check licenses
- [ ] Check generated docs
- [ ] Check vendored content

## Why it matters

Open-source readiness is not only about code quality. It also protects people, contracts, infrastructure, and trust.
