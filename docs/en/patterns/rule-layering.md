# Rule Layering

Layer rules so each one has a clear job.

- **Global rules** define personal or organization-wide defaults.
- **Repository rules** capture project-wide constraints and workflows.
- **Domain rules** explain specialized areas such as ML, docs, or deployment.
- **Task-specific rules** narrow the scope for a single change.
- **Tool-specific adapters** reshape the same intent for each AI tool.

Layering reduces conflicts and makes it easier to update the source of truth.
