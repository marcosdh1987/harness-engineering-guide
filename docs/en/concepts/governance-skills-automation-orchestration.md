# Governance, Skills, Automation, and Orchestration

## Governance

Governance defines durable policies and architectural constraints. Examples include open-source safety rules, review requirements, dependency policies, and documentation standards.

## Operational skills

Operational skills package repeatable task capabilities. A skill might guide an agent through updating documentation, validating a small change, or preparing a safe public example.

## Automation

Automation performs reliable mechanical checks. Common examples are CI builds, formatting, tests, link validation, and sync checks for generated artifacts.

## Orchestration

Orchestration connects the levels into a working method. A plan-first workflow, focused edits, validation gates, and explicit review discipline help AI tools behave like part of a system instead of isolated assistants.
