# Context Engineering

Context quality strongly affects AI-assisted development.

## What good context looks like

Good context includes clear repository structure, useful documentation, durable rules, concrete examples, and tests that show expected behavior.

## Why stale context is dangerous

Context can drift when documentation, generated rules, examples, and automation stop matching the real repository. AI tools then make confident but outdated decisions.

## How to reduce ambiguity

- Keep docs close to the code or workflow they describe.
- Prefer examples that match current practice.
- Remove or update stale instructions.
- Make validation steps easy to discover and run.
