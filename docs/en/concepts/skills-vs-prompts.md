# Skills vs Prompts

## One-off prompts

One-off prompts solve a local need in a single interaction. They are flexible, but they disappear quickly.

## Persistent instructions

Persistent instructions define standing expectations. They are useful for stable policies, style rules, and guardrails.

## Skills

Skills package repeatable work: a trigger, scope, inputs, outputs, validation, and examples.

## Agents

Agents execute tasks with tools and state. They can apply skills, follow instructions, and work through a plan.

## Workflows

Workflows define the order of work across people, tools, and validation gates.

## Adapters

Adapters translate the same source rules into the shapes expected by different tools, so the harness stays consistent without duplicating intent.
