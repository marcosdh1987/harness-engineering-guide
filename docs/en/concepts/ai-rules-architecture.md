# AI Rules Architecture

A practical harness can be described as four levels.

1. **Governance** — durable policies, architectural constraints, security boundaries, and public-sharing rules.
2. **Operational Skills** — repeatable capabilities for common tasks such as editing docs, running validation, or preparing releases.
3. **Automation** — CI jobs, formatting, tests, sync checks, and other mechanical gates.
4. **Orchestration** — plan-first workflows that decide when to apply rules, skills, and automation.

The four levels should reinforce each other. Governance sets boundaries, skills make common work easier, automation verifies outcomes, and orchestration keeps execution disciplined.
