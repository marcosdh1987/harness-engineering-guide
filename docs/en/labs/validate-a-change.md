# Lab: Validate a Change

## Goal

Practice validating an AI-assisted edit before sharing it.

## Scenario

A small documentation update looks correct, but you want a repeatable safety check.

## Steps

1. Review the task and changed files.
2. Run the documented validation command.
3. Inspect the diff for unrelated edits.
4. Confirm links, examples, and commands still make sense.

## Expected output

A reviewed change with evidence that the documented checks passed.

## Reflection questions

- Which checks are automated?
- Which checks still require human review?

## Safety notes

Validation should include secrets scanning and a review for accidental private content.
