# Lab: Publish Safe Docs

## Goal

Prepare documentation for a public open-source release.

## Scenario

Your team wants to publish internal notes as public learning material.

## Steps

1. Review the open-source readiness checklist.
2. Remove sensitive names, links, and screenshots.
3. Confirm examples are safe to publish.
4. Build the docs and review the generated site.

## Expected output

A documentation set that is useful, public-safe, and ready for review.

## Reflection questions

- What information had to be generalized?
- What checks should become automated next?

## Safety notes

Treat unknown data as sensitive until it is explicitly cleared for public release.
