# Lab: Create a Skill

## Goal

Package a repeated engineering task as a durable skill.

## Scenario

You often need the same sequence for updating docs and validating the result.

## Steps

1. Name the skill.
2. Define its trigger and scope.
3. List inputs and expected outputs.
4. Add validation and failure modes.
5. Include a small example.

## Expected output

A reusable skill that different tasks can invoke consistently.

## Reflection questions

- What part of the task is stable?
- What part still needs human judgment?

## Safety notes

Keep the skill public-safe and avoid encoding sensitive project context.
