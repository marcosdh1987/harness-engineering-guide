# Lab: Create a Rule

## Goal

Draft a durable repository rule for AI-assisted work.

## Scenario

Your team repeats the same review reminder on every documentation change.

## Steps

1. Identify the repeated expectation.
2. Write the rule in clear, durable language.
3. Place it where tools and humans can find it.
4. Add one example and one validation step.

## Expected output

A rule that is easy to reuse without rewriting it in every prompt.

## Reflection questions

- Is the rule specific enough to change behavior?
- Is it broad enough to survive many tasks?

## Safety notes

Do not include secrets, credentials, private URLs, or client-specific instructions.
